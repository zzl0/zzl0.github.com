    • Requirements

    以下几个场景需要大量 metric 记录和查询：

  • 记录每个 nurl 的响应时间
  • 记录每个 normalized sql 的响应时间
  • 记录每个 pidl key 的历史状态

    • 目前的 graphite 系统在数十万 metrics 时写入 IO 已经成为瓶颈，一个 key 一个文件的设计在通配符查询时也会带来大量文件系统操作。不适合此场景。
